# Security & Disclosure Policy

## 1. Purpose and Scope
This document outlines the security protocols and vulnerability disclosure procedures for the codebase repository associated with my personal portfolio website. Despite its nature as a static, client-side application, safeguarding the integrity and security of the code is of paramount importance.

## 2. Reporting a Vulnerability
Any individual or entity who discovers a potential security vulnerability, structural flaw, or systemic defect within this repository is formally requested to report said findings immediately. 

Disclosures must be submitted via one of the following channels:
- By initiating a GitHub Issue within this repository.
- Via direct electronic communication using the contact details provided in my GitHub profile.

**Note:** Public disclosure or exploitation of any vulnerability prior to official resolution and authorization is strictly prohibited.

## 3. Supported Versions and Maintenance
Maintenance, active support, and security patches are exclusively provided for the most recent iteration of the codebase residing on the `main` branch. 

All prior releases, historical commits, forks, or detached branches are expressly excluded from any security guarantees, warranties, or active maintenance. Users cloning or forking this repository do so at their own risk.
